Enum - UriParseHints

Module
juptune.http.uri
enum UriParseHints : int
{
  // 
  none = 0,
  // e.g. "scheme://authority/path?query#fragment" 
  isAbsolute = 1 << 0,
  // e.g. "//authority/path?query#fragment" 
  isNetworkReference = 1 << 1,
  // e.g. "authority/path?query#fragment", see `UriParseRules.allowUriSuffix` 
  isUriSuffix = 1 << 2,
  // e.g. "host:port" 
  authorityHasPort = 1 << 3,
  // e.g. "user:info@host" 
  authorityHasUserInfo = 1 << 4,
  // e.g. "[::1]" 
  authorityHostIsIpv6 = 1 << 5,
  // e.g. "127.0.0.1" 
  authorityHostIsIpv4 = 1 << 6,
  // e.g. "localhost" 
  authorityHostIsDomain = 1 << 7,
  // e.g. "/some/path" 
  pathIsAbsolute = 1 << 8,
  // e.g. "some/path" 
  pathIsRootless = 1 << 9,
  // e.g. "" 
  pathIsEmpty = 1 << 10,
  // e.g. "abc:123/abd" 
  pathHasStartColon = 1 << 11,
  // e.g. "" 
  queryIsEmpty = 1 << 12,
  // e.g. "" 
  fragmentIsEmpty = 1 << 13,
  // e.g. "user%40info@host" 
  percentEncodedUserInfo = 1 << 14,
  // e.g. "[::1%25%3a%3a1]" 
  percentEncodedHost = 1 << 15,
  // e.g. "/some%2fpath" 
  percentEncodedPath = 1 << 16,
  // e.g. "?some%3fquery" 
  percentEncodedQuery = 1 << 17,
  // e.g. "#some%23fragment" 
  percentEncodedFragment = 1 << 18,
}

About

A bitmask of hints that can be used to determine the structure of a URI.

Users should not set these values themselves as it may cause incorrect behaviour during parsing. Only parsing functions should set these values.

Since the end result of a URI is essentially a set of strings, certain aspects of the URI are lost or ambiguous, so the parser returns these hints to help the caller determine the exact nature of the URI.
